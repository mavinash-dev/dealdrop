## Why Open Source?

1. **Free marketing** — GitHub is a discovery platform. A well-maintained repo gets stars, forks, and attention
2. **Free contributors** — Developers add features you don't have time to build
3. **Trust signal** — Users and brands trust open source products more
4. **Investor appeal** — GitHub stars = social proof for early-stage startups
5. **Community moat** — Harder for competitors to copy a community

---

## License Choice

Use **MIT License** — the most permissive and contributor-friendly open source license.

```
MIT License — allows anyone to use, copy, modify, and distribute
the code freely, even for commercial use, as long as they include
the original license notice.
```

> ⚠️ Keep your Supabase keys, affiliate API keys, and any secret configs in `.env` files — never commit these to GitHub.

## How to Attract Contributors

### Week 1 — Setup
- Add detailed `README.md` with live demo link, screenshots, and setup guide
- Add `CONTRIBUTING.md` with clear steps to run locally
- Create GitHub Issues with labels: `good first issue`, `help wanted`, `bug`, `enhancement`

### Week 2–4 — Outreach
- Post on: dev.to, hashnode.dev, Reddit r/opensource, r/reactjs
- Tweet with: #opensource #buildinpublic #100DaysOfCode
- Post in Discord servers: Reactiflux, Supabase Discord, IndiaDevs

### Month 2+ — Community
- Respond to every PR and Issue within 24 hours
- Give contributors credit in README and CHANGELOG
- Create a `Contributors` section in README with avatars
- Monthly community update posts on Twitter/LinkedIn

## Issue Templates

### Good First Issues (easy, beginner-friendly):
- Add a new store logo to the store list
- Fix a typo in README
- Add a new coupon category/tag
- Improve mobile responsiveness of a component
- Add unit test for a utility function

### Feature Issues (for experienced contributors):
- Build expiry reminder system
- Add browser extension for auto-saving coupons
- Integrate a new affiliate API
- Build admin dashboard for deal moderation
